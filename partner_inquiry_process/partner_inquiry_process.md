# 파트너 문의 처리 프로세스

## 프로세스 개요

파트너의 문의가 젠데스크로 유입되어 처리 완료까지의 전체 프로세스

## 프로세스 플로우차트

```mermaid
flowchart TD
    Start([시작: 파트너 문의 발생])
    
    %% 문의 유입 경로
    subgraph Inbound["문의 유입"]
        A1[이메일 접수<br/>guide@myrealtrip.com]
        A2[마리트 홈페이지<br/>문의하기 접수]
    end
    
    %% 젠데스크 티켓 생성
    subgraph Zendesk["젠데스크 티켓 생성"]
        B1[티켓 생성<br/>메일 형식 또는 메시징 형식]
        B2[파트너지원 처리중 상태]
    end
    
    %% 문의 유형 분기
    CondType{문의 유형 확인}
    
    %% 상품 수정 요청
    subgraph Product["상품 정보 수정"]
        C1[수정 가능 영역 확인<br/>대표사진, 불포함사항<br/>기타 예약정보, 취소환불규정]
        C2[파트너 페이지 권한 확인]
    end
    
    CondEdit{파트너가<br/>직접 수정 가능?}
    
    subgraph SelfEdit["파트너 직접 처리"]
        D1[마리트 보기 권한 제공]
        D2[파트너가 직접 수정]
    end
    
    subgraph ExceptionEdit["예외 처리 필요"]
        E1[예외 환불 규정 요청]
        E2[담당 PO에게 전달]
    end
    
    %% 환불 요청
    subgraph Refund["환불 요청 처리"]
        F1[부분환불 요청 확인]
        F2[전액환불 요청 확인<br/>여행완료 건]
        F3[환불 처리 진행]
    end
    
    %% 양수양도
    subgraph Transfer["양수양도 처리"]
        G1[양수양도 요청 확인]
        G2[양수양도 처리 진행]
    end
    
    %% 정보 변경
    subgraph InfoChange["정보 변경 요청"]
        H1[계좌 변경 요청]
        H2[각종 정보 변경 요청]
        H3[사업자 주소 변경 불가 안내]
    end
    
    %% 상품 페이지 정보
    subgraph PageInfo["상품 페이지 정보"]
        I1[파트너 정보 노출 관련]
        I2[상품 페이지 수정 요청]
    end
    
    %% 시스템 관련
    subgraph System["시스템 관련"]
        J1[시스템 오류 확인]
        J2[개발팀 또는 관련팀 전달]
    end
    
    %% 상품 노출
    subgraph Exposure["상품 노출 관련"]
        K1[상품 노출 상태 확인]
        K2[노출 설정 조정]
    end
    
    %% AI 후기
    subgraph AIReview["AI 후기 요약"]
        L1[AI 후기 요약 이슈 확인]
        L2[고도화 미완료로 삭제 처리]
    end
    
    %% 핑퐁 케이스
    subgraph PingPong["핑퐁 케이스"]
        M1[추가 정보 요청]
        M2[파트너 응답 대기]
        M3[재확인 및 처리]
    end
    
    %% 타 부서 전달
    subgraph OtherDept["타 부서 전달"]
        N1[정산 관련<br/>매출관리팀, 자금팀]
        N2[사업실 관련]
        N3[서비스운영실 관련]
        N4[담당 PO 전달]
    end
    
    %% 처리 완료
    subgraph Complete["문의 처리 종료"]
        O1[처리 완료 안내]
        O2[티켓 종료]
    end
    
    End([완료: 문의 처리 종료])
    
    %% 흐름 연결
    Start --> A1
    Start --> A2
    A1 --> B1
    A2 --> B1
    B1 --> B2
    B2 --> CondType
    
    %% 상품 수정 경로
    CondType -->|상품 정보 수정| C1
    C1 --> C2
    C2 --> CondEdit
    CondEdit -->|가능| D1
    CondEdit -->|불가| E1
    D1 --> D2
    D2 --> O1
    E1 --> E2
    E2 --> O1
    
    %% 환불 요청 경로
    CondType -->|환불 요청| F1
    F1 --> F2
    F2 --> F3
    F3 --> O1
    
    %% 양수양도 경로
    CondType -->|양수양도| G1
    G1 --> G2
    G2 --> O1
    
    %% 정보 변경 경로
    CondType -->|정보 변경| H1
    H1 --> H2
    H2 --> H3
    H3 --> O1
    
    %% 상품 페이지 경로
    CondType -->|상품 페이지| I1
    I1 --> I2
    I2 --> O1
    
    %% 시스템 경로
    CondType -->|시스템 관련| J1
    J1 --> J2
    J2 --> O1
    
    %% 상품 노출 경로
    CondType -->|상품 노출| K1
    K1 --> K2
    K2 --> O1
    
    %% AI 후기 경로
    CondType -->|AI 후기 요약| L1
    L1 --> L2
    L2 --> O1
    
    %% 핑퐁 케이스 경로
    CondType -->|추가 정보 필요| M1
    M1 --> M2
    M2 --> M3
    M3 --> O1
    
    %% 타 부서 전달 경로
    CondType -->|타 부서 관련| N1
    CondType -->|타 부서 관련| N2
    CondType -->|타 부서 관련| N3
    CondType -->|PO 문의| N4
    N1 --> O1
    N2 --> O1
    N3 --> O1
    N4 --> O1
    
    %% 완료
    O1 --> O2
    O2 --> End
    
    %% 스타일 정의
    classDef default fill:#FFF9C4,stroke:#FFD54F,stroke-width:2px,color:#000
```

## 주요 문의 유형

### 1. 상품 정보 수정
- **파트너 직접 수정 가능**: 대표사진, 불포함사항, 기타 예약정보, 취소환불규정
- **예외 처리 필요**: 예외 환불 규정 희망 시 PO 전달

### 2. 환불 요청
- 부분환불 요청
- 전액환불 요청 (여행완료 건)

### 3. 양수양도
- 파트너 간 권리 이전 처리

### 4. 정보 변경
- 계좌 변경 요청
- 각종 정보 변경 요청
- **주의**: 사업자 주소 변경 불가

### 5. 상품 페이지
- 파트너 정보 노출 관련
- 상품 페이지 수정 요청

### 6. 시스템 관련
- 시스템 오류 확인
- 개발팀 또는 관련팀 전달

### 7. 상품 노출 관련
- 상품 노출 상태 확인
- 노출 설정 조정

### 8. AI 후기 요약
- AI 후기 요약 이슈
- **현재**: 고도화 미완료로 삭제 처리

### 9. 핑퐁 케이스
- 추가 정보 요청
- 파트너 응답 대기
- 재확인 및 처리

### 10. 타 부서 전달
- **정산 관련**: 매출관리팀, 자금팀
- **사업실**
- **서비스운영실**
- **담당 PO**

## 문의 유입 경로

### 이메일
- guide@myrealtrip.com 으로 접수
- 젠데스크 티켓 자동 생성

### 홈페이지
- 마리트 홈페이지 문의하기
- 젠데스크 티켓 자동 생성

## 티켓 형식

- **메일 형식**: 일반 이메일 스레드
- **메시징 형식**: 채팅 형태의 대화

## 처리 흐름

```
문의 유입 → 젠데스크 티켓 생성 → 파트너지원 처리중 → 유형 분류 → 처리 → 완료
```
